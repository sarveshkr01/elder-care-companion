[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture diagram to see how it works under the hood. 
Every query first passes through the Security Checkpoint node. If it detects a safety violation or prompt injection, it routes immediately to the Security Failure node to block access. If the input is verified as safe, it is cleaned of PII and passed to the Orchestrator. 
The Orchestrator acts as the lead coordinator, using Agent Tools to delegate specific tasks to either the Medication Agent or the Visit Agent. These specialized sub-agents interact with our local MCP Server, which reads and writes data to structured JSON databases. 
If an action requires caregiver approval, the workflow pauses at the Human Caregiver Approval node using a RequestInput state, continuing only once the caregiver grants permission, and finally returning a status update via the Final Response node.
